JobDocs API Specifications

Note - This is a specification of what needs to be built not of an existing API. If any methods need to be adjusted to suit actual usage during development feel free to do so, but please let me know when changes occurred so I can document them.

Sample Root URL

For this release, the root paths of all methods should begin with 'api/v1' like this:
https://jdmobileapp.azurewebsites.net/api/v1/v1/

An example to get all jobs whould therefore be:
https://jdmobileapp.azurewebsites.net/api/v1/v1/jobs
