Entity Descriptions

These are the basic types of resources that can be found in the JobDocs domain. Please refer to the Data model project in the solution for the attributes of these entities, both EF and DTO objects exist. Please note that some of these may not make sense to provide to an API consumer directly and would have very different DTOs unrelated to the entity model.

  • Job - Identifies the occupation the user undertakes in order to provide realistic jobroles, job resources and categories to the user. (In our initial release there is only "Nursing", but the data model could support more someday if needed.)
  • JobRole - A subspecialization of the Job. Given our one known Job of "Nursing" Job Roles include such thigns as "RN" or "LPN".
  • JobResources - A Job Resource is a link to websites that are of relevance to a given Job or Job Role Category - A Category provides a logical grouping for the users documents. Categories are unique to a job, but user defined categories can be added.
  • CategoryItem - A Category Item is an individual document stored by a user.
  • CategoryItemImage - A Category Item Image is a picture of the document referred to by the CategoryItem. A Category Item will contain one or more CategoryItemImages
  • ApplicationUser - Contains both the profile of the user and information on their identity in the system.
  • UserFeedback - Contains a record of any feedback the user provided via the mobile app
  • UserJob - Tracks associations between ApplicationUsers and their Jobs, Job Roles and Categories.
