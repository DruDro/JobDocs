Security Considerations

Please note that each entity has various security settings to restirct access, there are a few patterns.

  • **Anonymous GETs, authenicated admin roles required for everything else **- This pertains to Jobs, JobRoles, and JobResources.
  • Anonymous GETs, authenticated admin roles can do anything while authenticated user roles can only affect their own data - Applies to Categories
  • Authentication always required, admins can modify user data but users can only modify their own data - Applies to CategoryItems, CategoryItemImages.

Users and User feedback are special case, see the API docs for those resources.
